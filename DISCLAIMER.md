## 四、V0.3 校准数据与隐私特别说明

V0.3 引入了动态校准功能。本功能会记录历次推理的内存使用数据，以提升上下文窗口估算的准确性。使用本功能即表示您理解并同意以下条款：

### 4.1 校准数据的准确性

校准数据（包括每次推理前后的内存快照、KV cache 字节数、处理的 token 数）会被存储到本地 CSV 文件中。**校准结果的准确性依赖于采集数据的质量和数量**：

- 采集次数不足 10 次时，校准结果处于低置信度阶段，估算仍偏保守。
- 校准是一个渐进过程，前 3-5 次的估算可能显著偏离实际情况。
- 校准引擎使用加权平均算法，仅根据历史数据趋势进行平滑推断，**不做任何因果关系判断或预测性分析**。

### 4.2 模型名的哈希处理

校准数据中的模型名将经过 **SHA256 哈希处理**（取前 16 位 hex 字符），并拼接本机 `/etc/machine-id` 的前 8 位作为盐值（salt）。这意味着：

- CSV 文件中存储的仅为模型名的哈希值，不包含原始模型名。
- 哈希值在不同机器之间不可关联——同一模型在不同机器上的哈希值因 salt 不同而不同。
- **但是**，在单台机器上，哈希值可以唯一标识一个模型（因为 salt 固定）。如果您对此有隐私顾虑，请定期运行 `--reset-calibration` 清空数据。

### 4.3 云服务器用户特别提醒

如果您在云服务器（包括但不限于 AWS EC2、阿里云 ECS、腾讯云 CVM 等）上使用本软件，请注意：

- 校准数据存储在 `~/.config/hawk-eye-mem/calibration/` 目录下。
- **临时租用的云服务器在销毁后，磁盘数据可能被后续租户恢复。**
- 销毁实例前，建议执行 `--reset-calibration` 或手动删除上述目录中的所有校准数据文件。
- 如果您使用共享或临时云环境，建议不要启用本软件的校准功能，或确保在使用结束后清空校准数据。

### 4.4 V0.3 状态机与紧急通道

V0.3 引入了基于时间的连续监控状态机（Normal/Warning/Critical 三态模型）和紧急快速通道。当系统可用内存低于 `总内存 × 10%` 且低于 `512MB` 时，状态机将立即切换到 Critical 状态并触发 `AbortSafely` 建议。

**此机制为软件层面的数学判定，不构成对硬件故障、内存泄漏或其他系统异常的物理检测或诊断。** 即使状态机未触发 Critical 状态，系统仍可能因各种不可预知的原因（如瞬时内存尖峰、内存碎片、硬件故障）发生内存耗尽或进程崩溃。

## 五、V0.4 新功能的特别说明

### 5.1 环境指纹

V0.4 引入了环境指纹功能。本功能会在首次运行时采集当前机器的硬件配置信息（CPU 型号/核心数、总内存容量、GPU 型号/显存、磁盘容量），生成 HMAC-SHA256 签名的唯一标识并存储在本地 `~/.config/hawk-eye-mem/environment/` 目录下。

**隐私说明：**
- 环境指纹**仅包含硬件规格元数据**（如"8核CPU、16GB内存、NVIDIA RTX 4090 24GB"），**不包含**文件内容、运行中的进程列表、网络数据、用户文件或任何个人身份信息。
- 指纹数据**不会自动上传或发送到任何远程服务器**。仅在用户主动启用 `--serve` 远程采集模式时，指纹信息才会作为 `/metrics` 或 `/full` 响应的一部分被传输。
- 每次运行软件时自动比对当前硬件与已保存的指纹。检测到变更后，软件在 JSON 输出的 `environment_change` 字段中提供描述性文本（如"内存已升级至 32GB，可以运行更大模型"），不会记录或上报环境变更事件。

**环境变更检测的局限性：**
- 指纹仅覆盖 CPU、内存、GPU、磁盘四项硬件规格。不检测内核版本、驱动版本、内核模块、系统配置等其他可能影响软件运行的环境因素。
- 指纹匹配是精确匹配——硬件规格的微小差异（如 CPU 频率微调、磁盘剩余空间变化）可能被识别为环境变更。
- **环境变更检测不能替代完整的部署测试。** 即使指纹未变，目标机器的软件环境（驱动、库版本、内核配置）仍可能与本机存在显著差异。

### 5.2 远程采集 HTTP 服务

V0.4 支持通过 `--serve` 启动轻量 HTTP 服务，暴露系统指标端点。**此功能将您的机器变为网络可访问的指标采集节点，具有以下安全考量：**

**安全责任：**
- HTTP 服务**默认仅绑定 `127.0.0.1`（本地回环地址）**。如果需要在网络上可访问，用户必须手动将 `--bind` 参数改为 `0.0.0.0` 或具体网卡 IP。
- 绑定到公网 IP 或 `0.0.0.0` 时，服务将暴露于网络。我们**强烈建议**：
  - 配置 `--api-key` 参数启用 API Key 认证
  - 仅在公司内部网络或 VPN 环境下暴露服务
  - 不在没有认证的公共网络上运行此服务
- **即使配置了 API Key，HTTP 传输是明文的。** API Key 和指标数据均未加密。对于敏感环境，请在 HTTPS 反向代理（如 Nginx、Caddy）之后运行。
- 服务的 `/metrics` 和 `/full` 端点**包含系统可用内存、CPU 负载、磁盘使用率等可能反映系统状态的敏感信息**。公开暴露可能导致安全信息泄露。
- 速率限制机制（每 IP 每秒 10 次请求）为软件层面的尽力而为措施。**不构成拒绝服务攻击的完全防御。**

### 5.3 趋势分析与历史数据

V0.4 引入了趋势分析功能。本功能会将历次运行的系统指标记录到历史数据文件（`~/.config/hawk-eye-mem/history/`），用于线性回归分析和趋势预测。

**数据说明：**
- 历史数据仅包含时间戳、内存/CPU/磁盘三个维度的聚合指标。**不包含**进程列表、文件路径、网络连接、用户身份或其他可识别信息。
- 趋势报告中的 urgency 评估是基于线性回归算法的**数学推断**，**不构成对系统未来状态的确定性预测**。
- 系统资源使用模式可能因周期性任务、定时作业或用户行为模式而呈现非线性的循环波动。线性回归无法捕捉此类周期性信号，可能导致趋势误判。
- 历史数据保留期限由配置文件中的 `[history].retention_days` 控制（默认 7 天）。超过保留期限的数据会在下次运行时自动清理。
- 如果不需要趋势分析功能，可以通过 `--clear-history` 随时清空所有历史数据。

### 5.4 并发度建议（`--suggest-concurrency`）

V0.4 引入了 `--suggest-concurrency` 功能，基于当前系统资源（可用内存、CPU 核心数、GPU 显存）为 AI Agent 框架提供建议的并发子 Agent 数量。

**此建议为粗粒度估算，具有以下局限性：**
- 建议基于**当前瞬时**的资源状态，不预测未来变化。如果其他程序随后启动并占用资源，实际可安全运行的并发数会减少。
- 如果用户未通过 `--task-memory` 指定每个子 Agent 的内存开销，系统使用默认值（1GB/Agent）。该默认值可能不适合您的特定工作负载。
- **并发建议不是对资源争用、死锁、性能退化或其他并发编程问题的解决方案或预防措施。**
- **因采纳并发建议而导致的 OOM、服务不稳定或性能下降，本软件不承担责任。**
