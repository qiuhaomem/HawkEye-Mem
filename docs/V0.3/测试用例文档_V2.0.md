## 二、核心适配：低内存环境压力判定修正

**背景**：V0.2 的 `critical` 触发条件包含 `available < 2GB`。在总内存仅 2GB 的云服务器上，系统启动后可用内存就低于此值，导致永久误报。

**修正方案**：当总内存 ≤ 4GB 时，使用百分比判定替代绝对值判定。

| 判定维度 | 通用阈值（总内存 > 4GB） | 低内存阈值（总内存 ≤ 4GB） |
|----------|------------------------|--------------------------|
| `critical` | available < 2GB 或 used > 92% | **used > 90%** |
| `high` | available < 4GB 或 used > 80% | **used > 70%** |
| `medium` | available < 8GB 或 used > 50% | **used > 40%** |
| `low` | 其余 | **used ≤ 40%** |
| **紧急快速通道** | **available < 512MB 或 used > 98%** | **同左（不受低内存模式影响）** |
