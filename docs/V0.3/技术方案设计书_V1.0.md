## 一、架构变更概述

V0.2 的核心架构：CLI 入口 → 部署评估引擎 / 实时监控引擎 → ResourceCollector → 各资源采集器。

V0.3 新增三个核心模块：

```
┌─────────────────────────────────────────────────────┐
│                   CLI 入口 (main.rs)                  │
│   + --calibration-stats / --reset-calibration        │
│   + --gpu-list                                       │
└──────────────┬──────────────────┬────────────────────┘
               │                  │
    ┌──────────▼──────────┐    ┌──▼────────────────────┐
    │   部署评估引擎       │    │   实时监控引擎         │
    │   AssessmentEngine  │    │   MonitoringEngine    │
    │   (--can-run)       │    │   (--json / --metric) │
    └─────────────────────┘    └──┬────────────────────┘
                                  │
              ┌───────────────────┼───────────────────┐
              │                   │                   │
    ┌─────────▼────────┐  ┌──────▼──────┐  ┌─────────▼────────┐
    │  状态机引擎       │  │  动态校准    │  │  ResourceCollector│
    │  StateMachine    │  │  Calibration │  │  (统一 trait)      │
    │  (--interval)    │  │  Engine      │  └─────────┬────────┘
    └──────────────────┘  └──────┬──────┘            │
                                 │        ┌──────────┼──────────┐
                                 │        │          │          │
                          ┌──────▼──┐ ┌──▼──┐ ┌────▼────┐ ┌───▼───┐
                          │  CSV    │ │Memory│ │Disk/CPU │ │ GPU   │
                          │  Store  │ │Col.  │ │Col.     │ │ Col.  │
                          └─────────┘ └─────┘ └─────────┘ └───────┘
```

**核心变更**：
1. 新增 `StateMachine` 引擎：基于时间窗口的压力状态转换。
2. 新增 `CalibrationEngine` + `CsvStore`：推理前后内存快照采集、校准数据持久化、动态修正估算参数。
3. GPU Collector 重大升级：NVIDIA 增强（NVML 直接绑定）、AMD 新增（ROCm）、Apple Silicon 新增（Metal API）。
4. 新增温度采集模块（CPU/GPU）和多 Agent 进程检测模块。


## 二、动态校准引擎

### 2.1 校准数据采集

**数据点计算**：
```rust
impl CalibrationEngine {
    pub fn record_inference(
        &self,
        before: &ResourceSnapshot,
        after: &ResourceSnapshot,
        tokens_processed: u64,
    ) -> Option<CalibrationPoint> {
        if tokens_processed == 0 { return None; }
        
        let before_avail = before.memory.as_ref()?.available_mb;
        let after_avail = after.memory.as_ref()?.available_mb;
        let delta_mb = before_avail.saturating_sub(after_avail);
        
        // 过滤噪声：内存变化 < 10MB 不记录
        if delta_mb < 10 { return None; }
        
        let bytes_per_token = (delta_mb as u64) * 1024 * 1024 / tokens_processed;
        
        Some(CalibrationPoint {
            timestamp: SystemTime::now(),
            bytes_per_token,
            tokens_processed,
        })
    }
}
```

### 2.2 校准数据存储（CsvStore）

**文件路径**：`~/.config/hawk-eye-mem/calibration.csv`

**格式**（CR-05：模型名哈希处理）：
```csv
timestamp,model_hash,bytes_per_token,tokens_processed
1716153600,a1b2c3d4,1850,4096
1716154200,a1b2c3d4,1920,8192
```

**哈希算法**：SHA256(model_name).to_hex()[..8]

**并发写入保护（CR-02）**：
```rust
impl CsvStore {
    pub fn append(&self, point: CalibrationPoint, model_name: &str) -> Result<()> {
        let file = OpenOptions::new()
            .create(true)
            .append(true)
            .open(&self.path)?;
        
        // 文件锁
        if let Err(_) = flock(&file, LockMode::Exclusive, Some(Duration::from_millis(100))) {
            // 锁获取失败，放弃写入不阻塞
            return Ok(());
        }
        
        let model_hash = sha256_hash(model_name, 8);
        writeln!(file, "{},{},{},{}", 
            point.timestamp, model_hash, point.bytes_per_token, point.tokens_processed)?;
        
        // 数据量上限：最多 100 条
        self.trim_if_needed(100, 50)?;
        Ok(())
    }
}
```

### 2.3 动态修正算法

```rust
impl CalibrationEngine {
    pub fn get_corrected_params(&self, model_name: &str) -> Option<(u64, f64)> {
        let points = self.store.read_by_model(model_name)?;
        if points.len() < 10 { return None; } // 样本数 < 10 回退保守公式
        
        // 加权平均：最新 10 次线性衰减权重
        let weighted = points.iter().rev().take(10).enumerate()
            .map(|(i, p)| (p.bytes_per_token, 1.0 - i as f64 * 0.1))
            .collect::<Vec<_>>();
        
        let total_weight: f64 = weighted.iter().map(|(_, w)| w).sum();
        let avg: f64 = weighted.iter().map(|(v, w)| *v as f64 * w).sum::<f64>() / total_weight;
        
        // 标准差
        let variance: f64 = weighted.iter()
            .map(|(v, w)| w * (*v as f64 - avg).powi(2))
            .sum::<f64>() / total_weight;
        let std_dev = variance.sqrt();
        let cv = std_dev / avg; // 变异系数
        
        // confidence 判定
        if cv <= 0.20 {
            Some((avg as u64, 0.10)) // 校准后安全边际降至 10%
        } else {
            None // 标准差过大，继续保守模式
        }
    }
}
```

**confidence 自动降级**：连续 3 次实测偏差 > 30% → 清空该模型数据，重新学习。

## 三、连续监控状态机

### 3.1 状态模型

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum MonitorState {
    Normal,
    Warning,
    Critical,
}

pub struct StateMachine {
    state: MonitorState,
    current_pressure: PressureLevel,
    state_entered_at: Instant,
}

impl StateMachine {
    pub fn update(&mut self, pressure: PressureLevel, now: Instant) -> Option<StateTransition> {
        let duration = now.duration_since(self.state_entered_at);
        
        match (&self.state, pressure) {
            // Normal → Warning：high/critical 持续 10 秒
            (MonitorState::Normal, PressureLevel::High | PressureLevel::Critical) 
                if duration >= Duration::from_secs(10) => {
                self.transition_to(MonitorState::Warning, now)
            }
            
            // Warning → Critical：critical 持续 15 秒
            (MonitorState::Warning, PressureLevel::Critical) 
                if duration >= Duration::from_secs(15) => {
                self.transition_to(MonitorState::Critical, now)
            }
            
            // Warning → Normal：medium/low 持续 10 秒
            (MonitorState::Warning, PressureLevel::Low | PressureLevel::Medium) 
                if duration >= Duration::from_secs(10) => {
                self.transition_to(MonitorState::Normal, now)
            }
            
            // Critical → Warning：high/medium/low 持续 15 秒
            (MonitorState::Critical, _) 
                if duration >= Duration::from_secs(15) && pressure < PressureLevel::Critical => {
                self.transition_to(MonitorState::Warning, now)
            }
            
            // 同状态内不变
            _ => None
        }
    }
}
```

**配置可调**（`[state_machine]`）：
```toml
[state_machine]
warning_seconds = 10
critical_seconds = 15
recovery_seconds = 10
```

**首次采集不延迟**：冷启动时若 pressure ≥ `high`，立即进入 `Warning`。

**状态重置**：每轮 `--count` 或 `--interval` 启动时，状态机归零。


## 四、GPU 完整支持

### 4.1 NVIDIA（增强：NVML 直接绑定）

```rust
#[cfg(feature = "gpu")]
pub struct NvmlCollector;

impl NvmlCollector {
    pub fn collect(&self) -> Result<Vec<GpuMetrics>> {
        let nvml = nvml_wrapper::Nvml::init()
            .or_else(|_| {
                eprintln!("Warning: NVML init failed, falling back to nvidia-smi");
                Self::fallback_nvidia_smi()
            })?;
        
        let count = nvml.device_count()?;
        let mut gpus = Vec::with_capacity(count as usize);
        
        for i in 0..count {
            let device = nvml.device_by_index(i)?;
            let name = device.name()?;
            let memory = device.memory_info()?;
            let temp = device.temperature(TemperatureSensor::Gpu)?;
            let power = device.power_usage()?;
            
            gpus.push(GpuMetrics {
                name,
                vram_total_mb: memory.total / (1024 * 1024),
                vram_used_mb: memory.used / (1024 * 1024),
                temperature_c: Some(temp as f64 / 1000.0),
                power_watts: Some(power as f64 / 1000.0),
                pressure: Self::calc_pressure(memory.free, memory.total),
            });
        }
        
        Ok(gpus)
    }
}
```

**unsafe 集中封装**：所有 NVML unsafe 集中在 `nvml.rs` 单一模块。

### 4.2 AMD（新增：rocm-smi 解析）

```rust
#[cfg(feature = "gpu")]
pub struct RocmCollector {
    rocm_smi_path: Option<String>,  // CR-03：配置文件可自定义路径
}

impl RocmCollector {
    pub fn collect(&self) -> Result<Vec<GpuMetrics>> {
        let path = self.rocm_smi_path.as_deref().unwrap_or("rocm-smi");
        let output = Command::new(path)
            .args(["--showid", "--showmeminfo", "vram", "--csv"])
            .output()
            .ok()
            .filter(|o| o.status.success())?;
        
        Self::parse_csv(&String::from_utf8_lossy(&output.stdout))
    }
}
```

### 4.4 多 GPU 与温度

- `system.gpu` 为数组，每 GPU 一个元素。
- 温度采集在各 Collector 内部完成，`GpuMetrics.temperature_c` 字段上报。
- 功耗仅 NVIDIA 支持，AMD/Apple 字段为 `null`。


## 五、温度/功耗监控

```rust
pub struct ThermalCollector;

impl ThermalCollector {
    #[cfg(target_os = "linux")]
    pub fn cpu_temp(&self) -> Option<f64> {
        let zones = std::fs::read_dir("/sys/class/thermal/")
            .ok()?;
        
        zones.filter_map(|entry| {
            let path = entry.ok()?.path();
            if !path.file_name()?.to_str()?.starts_with("thermal_zone") {
                return None;
            }
            let temp_str = std::fs::read_to_string(path.join("temp")).ok()?;
            let temp: f64 = temp_str.trim().parse().ok()?;
            Some(temp / 1000.0) // millidegree → degree
        })
        .max_by(|a, b| a.partial_cmp(b).unwrap())
    }
    
    #[cfg(target_os = "macos")]
    pub fn cpu_temp(&self) -> Option<f64> {
        // 通过 pmset -g thermlog 或 SMC 估算
        // 不可用时静默返回 None
        None
    }
}

pub struct ThermalMetrics {
    pub cpu_temp_c: Option<f64>,
    pub gpu_temps_c: Vec<Option<f64>>,
    pub throttle_warning: bool,
}

fn calc_pressure(temp: Option<f64>) -> Option<ThermalPressure> {
    temp.map(|t| match t {
        t if t < 80.0 => ThermalPressure::Normal,
        t if t < 95.0 => ThermalPressure::Warning,
        _ => ThermalPressure::Critical,
    })
}
```

**W4 裁剪评估**：若 P0 模块已占满工期，温度模块只采集不预警（`thermal_action` 延后 V0.4）。


## 六、多 Agent 资源感知

```rust
pub struct MultiAgentDetector {
    known_processes: Vec<String>,  // 内置 + 用户配置
}

impl MultiAgentDetector {
    pub fn detect(&self) -> Result<Vec<AgentProcess>> {
        let mut agents = Vec::new();
        
        #[cfg(target_os = "linux")]
        {
            for entry in std::fs::read_dir("/proc")? {
                let path = entry?.path();
                if !path.is_dir() { continue; }
                
                let pid_str = path.file_name()?.to_str()?;
                let pid: u32 = pid_str.parse().ok()?;
                
                // CR-06：只读取 comm，不读取 cmdline
                let comm = std::fs::read_to_string(path.join("comm")).ok()?;
                let name = comm.trim().to_string();
                
                if self.known_processes.contains(&name) {
                    let mem = self.get_process_memory(pid)?;
                    let cpu = self.get_process_cpu(pid)?;
                    agents.push(AgentProcess { name, pid, memory_rss_mb: mem, cpu_percent: cpu });
                }
            }
        }
        
        // 全局争抢预警
        if agents.iter().map(|a| a.memory_rss_mb).sum::<u64>() > total_memory * 0.8 {
            // global_pressure = "high"
        }
        
        Ok(agents)
    }
}
```

**约束**：只取进程名和内存，不读取 `cmdline`（CR-06）。无权限时 `multi_agent` 字段为 `null`。

## 八、关键风险与应对

| 风险 | 等级 | 应对 |
|------|------|------|
| `nvml-wrapper` 与 musl 静态链接兼容性 | 高 | V0.2 已验证通过，V0.3 继续监控 |

| 校准算法极端场景修正过度 | 中 | 加权平均 + 降级机制兜底 |
| AMD GPU 测试环境缺失 | 中 | 标记"需社区反馈"，功能可用但不承诺稳定性 |
