## 二、问题清单

### 🔴 致命级（必须修复）

| 编号 | 问题 | 影响 | 修复方案 |
|------|------|------|----------|
| **C1** | `PressureLevel` `#[serde(rename_all)]` 与 Action、Confidence 对齐 | 序列化输出一致性问题 | 已确认三枚举均使用 `#[serde(rename_all = "snake_case")]` |
| **C2** | macOS `sysctl_val` C 字符串 UB 风险 | macOS 平台潜在崩溃 | 当前使用 `name.as_ptr() as *const i8`，已改为 `CString::new(name)` |
| **C3** | macOS 可用内存未含 purgeable 页面 | 可用内存虚高，Agent 可能 OOM | 新增 `vm_page_purgeable_count`，purgeable 打 50% 折扣 |

### 🟠 严重级（强烈建议修复）

| 编号 | 问题 | 修复方案 |
|------|------|----------|
| **S1** | `EstimationEngine` 未处理 `available_mb = 0` | 添加守卫：`if available_mb < 256 { return (0, ...) }` |
| **S2** | Linux 老内核（< 3.14）无 `MemAvailable` | 添加 fallback 估算 |
| **S3** | `--metric` 错误信息确认写入 stderr | 已确认 `anyhow::Error` 默认走 stderr |
| **S4** | 无内存压力连续状态判断 | v0.1 暂不引入，文档说明 |

### 🟡 警告级（最佳实践）

| 编号 | 建议 |
|------|------|
| **W1** | 考虑使用 `std::time::SystemTime` 替代 `chrono` 以减少依赖（30+ 个间接依赖） |
| **W2** | `dirs::home_dir()` 在容器中返回 `None` 时需优雅降级 |
| **W3** | 建议 `src/main.rs` 和 `src/collector/linux.rs` 添加 `#![deny(unsafe_code)]` |
| **W4** | 需补充 `CONTRIBUTING.md`（含 AI 辅助编码声明，法务要求） |
