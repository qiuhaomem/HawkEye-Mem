## 应用端 1：AI Agent 运行时调用
这是秋毫mem的主战场。Agent在推理前、推理中或接收到内存相关指令时调用。

| 一级-应用端 | 二级-模块 | 三级-功能 | 四级-操作 | 五级-逻辑与约束 |
|---|---|---|---|---|
| **AI Agent 运行时调用** | **CLI 命令接口** | **JSON 结构化输出** | 执行 `hawk-eye-mem --json` | 输出必须为合法JSON，写入stdout；所有字段见接口规范v0.1；退出码0为成功，非0为失败 |
| | | | 执行 `hawk-eye-mem --json --interval 5` | 每5秒采集一次，输出为JSON Lines格式，每行一个完整JSON对象；首次采集立即执行不等待；**注意：Agent 集成仅使用单次调用模式（`--json` 无 `--interval`），`--interval` 模式定位为"开发者调试与脚本集成"功能** |
| | | **配置文件加载** | 自动查找配置文件 | 查找优先级（从高到低）：① `--config <path>` CLI参数 ② `HAWKEYE_MEM_CONFIG` 环境变量 ③ `$HOME/.config/hawk-eye-mem/config.toml`；所有路径查找失败时，静默回退默认值，不报错；文件存在但格式错误时，输出stderr警告并回退默认值 |
| | | **返回码规范** | 正常退出 | 返回码 0 |
| | | | 内存采集失败 | 返回码 1，stderr输出错误原因 |
| | | | 配置文件解析失败 | 返回码 0，但stderr输出警告，使用默认值继续 |
| | **指标估算引擎** | **safe context window 计算** | 使用保守公式 `floor(available_mb * 1024^2 / (bytes_per_token * (1 + margin)))` | `bytes_per_token` 默认2048；`margin` 默认30%；若用户配置了自定义值，则使用自定义值，并将 `confidence` 设为 `calibrated` |
| | | **confidence 判定（评审整改①）** | 判断估算可信度 | 改为两档制：用户未配置模型参数 → `"conservative"`；用户已配置或动态校准 → `"calibrated"`。**即使 `conservative`，Agent 仍可采纳建议**（安全边际30%已覆盖常见场景），无需等待用户配置 |
| | | **action 建议生成（评审整改②）** | 根据 `pressure` 和阈值映射 | `low` → `ok`；`medium` → `monitor`；`high` → `reduce_context`；`critical` → `abort_safely`（原 `stop_and_dump` 更名）；阈值表见第四节 |
| | | **reason 文案生成** | 拼接自然语言原因 | 必须包含：当前可用内存数值、判定依据、置信度声明；英文输出，不超过200字符 |
| | | **suggestion 引导字段（评审整改① 新增）** | 当 `confidence` 为 `conservative` 时 | 在 JSON 输出的 `agent_guidance` 中新增 `suggestion` 字段，固定值：`"Consider configuring your model parameters for more precise memory estimation. Run 'hawk-eye-mem --init-config' to get started."` |
| | **MCP 协议适配** | **Tool 注册（评审整改③）** | 作为MCP Tool暴露 `check_memory` | 输入：无必需参数，**不接受 `interval` 参数**（Agent 需要持续监控时，应由 Agent 自身的循环逻辑多次调用 Tool，而非由秋毫mem 内部循环）；输出：完整JSON结构；通信层：stdio |
| | | **Tool 调用** | Agent 通过MCP客户端调用 `check_memory` | 返回结构与 `--json` 一致；若MCP传输层断开，秋毫mem进程优雅退出 |

## 应用端 2：人类开发者调试
开发者在终端直接使用，用于快速排查或脚本集成。

| 一级-应用端 | 二级-模块 | 三级-功能 | 四级-操作 | 五级-逻辑与约束 |
|---|---|---|---|---|
| **人类开发者调试** | **CLI 命令接口** | **人类可读输出** | 执行 `hawk-eye-mem`（无参数） | 彩色终端输出，包含：总内存、可用内存、已用百分比、压力水位、建议上下文窗口、action建议；不输出原始JSON |
| | | | 执行 `hawk-eye-mem --count 3` | 执行3次采集后自动退出；适用于脚本化的快速检查 |
| | | **自定义配置路径** | 执行 `hawk-eye-mem --config /custom/path/config.toml --json` | 使用指定路径的配置文件覆盖默认查找逻辑 |
| | | **帮助信息** | 执行 `hawk-eye-mem --help` | 列出所有参数、用法示例、配置文件路径（含 `--config` 和环境变量 `HAWKEYE_MEM_CONFIG`）、项目链接 |
| | | **版本信息** | 执行 `hawk-eye-mem --version` | 输出版本号（如 `0.1.0`）、commit hash、构建日期 |
| | **配置文件管理** | **生成默认配置** | 执行 `hawk-eye-mem --init-config` | 在 `$HOME/.config/hawk-eye-mem/` 下生成带注释的 `config.toml` 模板 |
| | **错误与诊断** | **权限错误提示** | 当无法读取系统内存文件时 | 输出明确错误信息："Permission denied: cannot read /proc/meminfo. Try running with elevated privileges." |
| | | **不支持平台提示** | 当在 Windows 上运行时 | 输出："Unsupported platform. HawkEye Mem currently supports Linux and macOS only." |

## 应用端 3：运维与监控集成
用于被脚本、监控系统或其他自动化工具消费。

| 一级-应用端 | 二级-模块 | 三级-功能 | 四级-操作 | 五级-逻辑与约束 |
|---|---|---|---|---|
| **运维与监控集成** | **CLI 命令接口** | **仅输出关键指标（评审整改⑤）** | 执行 `hawk-eye-mem --metric available_mb`（成功时） | stdout 严格输出纯数值（如 `5800`），无额外字符；适用于shell管道 |
| | | | 执行 `hawk-eye-mem --metric available_mb`（失败时） | stdout 输出空字符串，stderr 输出错误信息，返回码为非零。**文档中给出安全调用示例**：`RESULT=$(hawk-eye-mem --metric available_mb) && if [ $? -ne 0 ]; then echo "Error: $RESULT" >&2; exit 1; fi` |
| | | | 执行 `hawk-eye-mem --metric pressure` | 仅输出字符串（如 `high`），无换行以外的字符 |
| | | **连续监控模式** | 执行 `hawk-eye-mem --json --interval 10 --count 0` | `--count 0` 表示无限循环，直到被 SIGINT 中断；输出JSON Lines流；**注意：此模式面向人类和管道消费，不作为Agent集成推荐方式** |
| | | **退出信号处理** | 接收到 SIGINT (Ctrl+C) | 优雅退出，输出一个包含 `"terminated": true` 的结束JSON行（若在JSON模式下） |

## 应用端 4：未来扩展预留
本期不做，但架构上必须留好接口。

| 一级-应用端 | 二级-模块 | 三级-功能 | 四级-操作 | 五级-逻辑与约束 |
|---|---|---|---|---|
| **未来扩展预留** | **MemoryCollector trait** | **平台扩展接口** | 定义 `trait MemoryCollector { fn collect() -> SystemMetrics; }` | 必须返回统一的 `SystemMetrics` 结构体；平台差异完全封装在实现内部 |
| | | **GPU Collector 预留** | 未来实现 `GpuCollector` 结构体 | 实现 `MemoryCollector` trait；在 `SystemMetrics` 中预留 `gpu_vram_mb` 字段（当前返回 `null`） |
| | **远程采集预留** | **远程数据源接口** | 未来支持 `--remote <url>` | 远程端点需返回与本地JSON同构的数据；通信协议优先考虑 HTTP/gRPC |
| | **插件化建议引擎** | **GuidanceGenerator trait** | 定义 `trait GuidanceGenerator { fn generate(metrics) -> AgentGuidance; }` | 默认实现为内置逻辑；预留 WASM 插件加载接口 |

## 应用端 5：安装与分发
确保用户能在5分钟内完成安装。

| 一级-应用端 | 二级-模块 | 三级-功能 | 四级-操作 | 五级-逻辑与约束 |
|---|---|---|---|---|
| **安装与分发** | **安装方式** | **单二进制分发** | 从 GitHub Releases 下载对应平台的二进制文件 | 支持 `x86_64-linux`, `aarch64-linux`, `x86_64-macos`, `aarch64-macos` |
| | | **包管理器安装** | `brew install hawk-eye-mem` | Homebrew formula 提交至官方仓库 |
| | | | `cargo install hawk-eye-mem` | crates.io 发布，源码编译 |
| | | **首次运行引导（评审整改⑥）** | 首次运行时 | **先输出简洁版免责声明，再输出快速引导**。免责声明控制在三行以内：`NOTICE: HawkEye Mem provides estimated suggestions only. No warranty of accuracy. Use at your own risk. See DISCLAIMER.md for full terms.`；首次运行检测机制：在 `$HOME/.config/hawk-eye-mem/` 下创建 `.onboarded` 标记文件；文件存在 → 跳过引导和声明；文件不存在 → 输出后创建 |
| | **文档** | **README.md** | 项目主页 | 必须包含：安装方法、快速开始、全部CLI参数说明、JSON输出示例、与Hermes集成示例 |
