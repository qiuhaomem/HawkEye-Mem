### 二、单元测试用例

#### 模块：估算引擎 (`src/engine/mod.rs`)

| 用例ID | 测试目标 | 前置条件 | 输入 | 预期结果 |
|--------|----------|----------|------|----------|
| **UT-EE-001** | 默认配置下，正常内存估算 | 无配置 | `available_mb=16384` | `estimated_tokens > 0`，`confidence = Conservative`，计算公式为 `16384*1024^2/2048*0.7` |
| **UT-EE-002** | 用户配置下，自定义参数估算 | 提供 `ModelConfig { bytes_per_token: 4096, margin: 20.0 }` | `available_mb=16384` | `estimated_tokens` 按 `16384*1024^2/4096*0.8` 计算，`confidence = Calibrated` |
| **UT-EE-003** | 极低内存边界测试 | 无配置 | `available_mb=512` | `estimated_tokens < 200_000`（保守估算不到200K） |
| **UT-EE-004** | 零内存或极低内存防护 | 无配置 | `available_mb=0` 或 `available_mb=128` | `estimated_tokens = 0`，不 panic，`confidence = Conservative` |
| **UT-EE-005** 🆕 | 中间水位边界值：4GB、8GB可用内存估算 | 无配置 | `available_mb=4096`、`available_mb=8192` | 4GB时 `estimated_tokens` 约1.4M（4096*1024^2/2048*0.7），8GB时约2.8M；均为 `Conservative`；4GB应接近 `High` 压力触发阈值 |

**说明（UT-EE-005）**：这是多方评审增补的高优先级用例。原文档覆盖了16GB（正常）和512MB（极低），但4GB和8GB恰好是 `reduce_context` 和 `monitor` 的临界区域，Agent 在这两个水位之间的决策是否平滑，需要提前验证。

**说明（IT-CLI-002B）**：多方评审增补。Agent 通过 MCP 调用时，返回值是字符串，多余空格或字符需要 Agent 额外处理。对人类无所谓，对 Agent 是隐患。

**说明（IT-CLI-009B）**：多方评审增补。Agent 消费 JSON Lines 时逐行解析，如果某行不合法，Agent 逻辑会中断。

#### 错误路径 JSON 输出验证 🆕

| 用例ID | 测试目标 | 前置条件 | 测试步骤 | 预期结果 |
|--------|----------|----------|----------|----------|
| **IT-ERR-001** 🔴 | 权限错误时输出合法错误 JSON | `/proc/meminfo` 不可读（模拟权限拒绝） | 运行 `hawk-eye-mem --json` | stdout 输出合法 JSON（含 `error` 顶级字段），退出码非 0；Agent 可以解析 `json.error` 获取错误描述 |
| **IT-ERR-002** 🔴 | 不支持的平台输出合法错误 JSON | 在未实现采集器的平台上运行 | 运行 `hawk-eye-mem --json` | stdout 输出合法 JSON（含 `error` 顶级字段），退出码非 0；Agent 无需特殊处理即可识别失败 |

**说明（IT-ERR-001/002）**：多方评审认定**致命级遗漏**。当 `--json` 因权限或平台不支持而失败时，如果不能输出合法 JSON，Agent 解析 JSON 的代码会直接崩溃。失败模式也必须是 Agent 可解析的结构化数据。

#### JSON 结构验证

| 用例ID | 测试目标 | 前置条件 | 测试步骤 | 预期结果 |
|--------|----------|----------|----------|----------|
| **IT-JSON-001** | 时间戳格式符合 RFC3339 | 运行 `--json` | 解析 `timestamp` 字段 | 可通过 `chrono::DateTime::parse_from_rfc3339` 解析 |
| **IT-JSON-002** | `collection_duration_ms` 为合理正数 | 同上 | 检查值 | `> 0.0` 且 `< 100.0`（通常 < 5ms） |
| **IT-JSON-003** | `used_percent` 精度为 1 位小数 | 同上 | 检查 `used_percent` 字符串 | 格式如 `42.5`，不多于 1 位小数 |
| **IT-JSON-004** | `estimated_safe_context_window` 为整数 | 同上 | 检查值类型 | JSON 数字，无小数点 |
| **IT-JSON-005** 🆕 | JSON Schema 关键字段存在性 | 运行 `--json` | 验证以下字段必须存在且类型正确：<br>`system.total_mb`（数字）、`system.used_mb`（数字）、`system.used_percent`（数字）、`system.available_mb`（数字）、`agent_guidance.action`（字符串）、`agent_guidance.pressure`（字符串）、`agent_guidance.estimated_safe_context_window`（数字） | 全部字段存在且类型正确；缺失任一字段视为失败 |

**说明（IT-JSON-005）**：多方评审增补。Agent 可能硬编码依赖特定字段名（如 `if json.agent_guidance.action == "reduce_context"`），如果某版本字段名称改动，Agent 会静默失败。Schema 一致性测试保障 Agent 端的字段稳定性。

### 四、手动验收测试

#### 场景1：Linux 高内存压力验证

| 项目 | 内容 |
|------|------|
| **环境** | Linux VM (8GB RAM) |
| **前置条件** | 加载一个 7B 模型 (如 Llama 3 8B)，使其占用约 5GB 内存 |
| **步骤** | 1. 运行 `hawk-eye-mem --json` <br> 2. 观察 `pressure` 和 `action` |
| **验收标准** | `pressure` 为 `high` 或 `critical`，`action` 为 `reduce_context` 或 `abort_safely`，`available_mb` < 4GB |
| **备注** | 应不会误报为 `low` 或 `ok` |

#### 场景2：macOS 正常使用不误报

| 项目 | 内容 |
|------|------|
| **环境** | MacBook Air 8GB (Apple Silicon) |
| **前置条件** | 关闭大部分应用，仅打开终端和浏览器 (轻度负载) |
| **步骤** | 1. 运行 `hawk-eye-mem --json` <br> 2. 记录 `pressure` |
| **验收标准** | `pressure` 为 `low` 或 `medium`，不会因 macOS 激进缓存而误报 `high` 或 `critical` |

#### 场景3：Agent 闭环验证（Hermes 集成）

| 项目 | 内容 |
|------|------|
| **环境** | Hermes 框架 + 秋毫mem Skill |
| **前置条件** | 系统内存充足，随后人为制造内存压力 (打开大型应用) |
| **步骤** | 1. 部署秋毫mem Skill 到 Hermes <br> 2. 让 Hermes 开始一个长时间推理任务，并自动调用 `check_memory` <br> 3. 逐步增加内存压力 <br> 4. 观察 Hermes 的反应 |
| **验收标准** | 1. 在内存充足时，Hermes 正常执行，`action=ok` <br> 2. 内存进入 `high` 时，Hermes **主动**告知用户"建议降低上下文" <br> 3. 内存进入 `critical` 时，Hermes 建议保存状态并中止推理 |

#### 场景4：首次运行引导与免责声明

| 项目 | 内容 |
|------|------|
| **环境** | 新安装的机器（或无 `.onboarded` 标记） |
| **前置条件** | 删除 `~/.config/hawk-eye-mem/.onboarded` |
| **步骤** | 运行 `hawk-eye-mem` (无参数) |
| **验收标准** | 1. **首先**输出免责声明 (必须包含 "No warranty" 和 "Use at your own risk") <br> 2. **然后**输出快速引导 (包含 `--json`、`--help` 提示) <br> 3. 再次运行，不再显示引导 |

#### 场景5：5 分钟上手测试

| 项目 | 内容 |
|------|------|
| **环境** | 全新 Linux/macOS 机器 |
| **前置条件** | 无 |
| **步骤** | 新用户仅凭 README 文档操作 |
| **验收标准** | 5 分钟内完成：下载二进制 → 运行 → 看懂 JSON 输出 → 修改配置文件 → 再次运行观察到 `confidence=calibrated` |
