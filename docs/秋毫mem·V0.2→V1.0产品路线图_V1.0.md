### 二、V0.2 —— 部署前评估（6周）

**里程碑**：

| 周次 | 里程碑 | 交付物 | 验收标准 |
|------|--------|--------|----------|
| W1 | 需求冻结 | V0.2 PRD | 产品经理+创始人签字 |
| W1-W2 | 磁盘Collector | `DiskCollector` trait实现 | 模型目录空间监控可用 |
| W2-W3 | CPU Collector | `CpuCollector` trait实现 | 负载+Agent进程CPU可用 |
| W3-W4 | `--can-run` 命令 | 部署评估逻辑+JSON输出 | 输入模型参数→输出verdict+safe_options |
| W4-W5 | GPU Collector（实验性） | NVIDIA NVML绑定 | `--features gpu` 编译，显存数据可用 |
| W5 | 模型参数库 | 预置8个主流模型配置 | 文档+代码内建 |
| W6 | 集成测试+发布 | v0.2.0二进制 | 全平台测试通过 |

**优先级**：P0=can-run+磁盘+CPU，P1=GPU实验性+模型参数库，P2=压力阈值校准v2。

**关键风险**：GPU NVML绑定可能遇到驱动版本兼容问题，若W5仍未解决，GPU降级为"仅nvidia-smi解析"或直接延后到V0.3。

### 三、V0.3/0.4 —— 精准自知 + 环境迁移（8-10周）

> **PMO说明**：V0.3和V0.4合并为一个版本。原因：动态校准（V0.3）和环境迁移（V0.4）在技术上共享"历史数据存储"和"趋势分析"两个基础模块，分开做会产生重复建设。合并后减少一次发布流程，社区只需升级一次。

**里程碑**：

| 周次 | 里程碑 | 交付物 | 验收标准 |
|------|--------|--------|----------|
| W1-W2 | 动态校准引擎 | 校准数据采集+自动修正 | `confidence`自动从conservative→calibrated |
| W2-W3 | GPU完整支持 | AMD ROCm + Apple Silicon Metal | 三大平台GPU均可监控 |
| W3-W4 | 连续监控状态机 | N次高压才触发机制 | 单点尖峰不误报 |
| W4-W5 | 多Agent感知 | 全局资源争抢检测 | 3个以上Agent同时运行时输出预警 |
| W5-W6 | 温度/功耗监控 | CPU+GPU温度墙预警 | 温度超阈值输出warning |
| W6-W7 | 环境指纹+迁移 | 环境指纹生成+变更检测 | 环境升级/降级时输出new_recommendation |
| W7-W8 | 远程采集+趋势 | HTTP/gRPC远程采集+7天历史 | 远程机器指标可用，趋势图可输出 |
| W8-W10 | 集成测试+Buffer | v0.4.0二进制 | 全平台+全场景测试 |

**商业化试点**：V0.4的远程采集功能作为企业版试点，`--remote`参数需商业许可证。核心CLI保持Apache-2.0。若社区对收费反馈负面，则撤回该限制，改为完全开源+捐赠模式。

### 四、V1.0 —— 完全体（8周）

**里程碑**：

| 周次 | 里程碑 | 交付物 | 验收标准 |
|------|--------|--------|----------|
| W1-W2 | 守护进程 `hawk-eye-memd` | 后台常驻+事件推送 | 持续监控，主动推送 |
| W2-W4 | 框架官方集成 | Hermes/LangChain/AutoGPT插件 | 一行代码集成 |
| W4-W5 | 资源预言机 | 基于历史趋势的预测模型 | 输出"预计X小时后到达临界" |
| W5-W6 | 多Agent协调 | 全局降级策略分配 | Agent A缩上下文、Agent B暂停 |
| W6-W7 | Web仪表盘 | 单机/多机可视化 | 实时监控+历史趋势+告警 |
| W7-W8 | 社区模型库上线 | 开放贡献+自动更新 | 覆盖50+主流模型 |

**商业化**：Web仪表盘作为SaaS产品独立运营。守护进程+框架集成+预言机保持开源。
