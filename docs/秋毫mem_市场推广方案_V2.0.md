### 一、战略定位

**品类**：Agent 环境感知层
**定位**：给 Agent 开个天眼
**核心叙事**：Agent 生下来是瞎的。它不知道自己在什么机器上，不知道还剩多少内存，不知道快把自己挤爆了。秋毫mem 就是 Agent 的天眼。

**品牌矩阵**：

| 元素 | 内容 |
|------|------|
| 项目名 | 秋毫mem / HawkEye Mem |
| 品类 | Agent 环境感知层 |
| Slogan (CN) | 秋毫mem —— Agent 的天眼 |
| Slogan (EN) | HawkEye Mem —— Eyes for your Agent |
| 一句话 | 给 Agent 开个天眼 |

**战略方针**：**V0.2 低调蓄力，V1.0 奇点引爆。** 不在功能可被快速复制的阶段暴露战略意图，在护城河完整时一举定义品类。


### 二、分阶段执行计划

#### 阶段一：V0.2 —— 低调蓄力（6周后发布）

**目标**：自然增长至 500 Stars，积累种子用户反馈，埋下品类定义种子。

**发布动作**：

| 动作 | 时间 | 负责人 |
|------|------|--------|
| GitHub Release + Release Notes | 周二下午 | 技术负责人 |
| 知乎/掘金技术博客《给 AI Agent 做了个部署前检测工具》 | 周三晚 | 市场运营 |
| README 嵌入用户反馈引导 Issue | 发布前 | 产品经理 |

**社区渗透（持续进行）**：

| 动作 | 原则 |
|------|------|
| 在 Ollama/llama.cpp 的 Issue 中帮助用户诊断环境问题 | 帮助优先，提及从轻。只在用户明确描述 OOM/下载失败时，以"我之前也遇到过"的口吻提及秋毫mem |
| 在 Reddit r/LocalLLaMA 回复 OOM 相关帖子 | 提供具体命令示例，不带比较性描述，标注开源身份 |

**不做的**：
- ❌ 不投 Reddit 主帖
- ❌ 不投 Hacker News Show HN
- ❌ 不主动联系 UP 主或媒体
- ❌ 不在任何渠道喊"品类定义"的口号

**成功指标**：
- GitHub Stars 自然增长至 500+
- V0.2 README 反馈 Issue 收到 10+ 条有效用户反馈
- 社区中自发出现 3 次以上"你可以试试秋毫mem"的非官方推荐


#### 阶段二：V0.3/0.4 —— 社区培养（V0.2 后 8-10 周）

**目标**：培养"秋毫mem = Agent 环境检测"的心智，积累动态校准数据，为 V1.0 准备生态基础。

**动作**：

| 动作 | 说明 |
|------|------|
| 继续社区渗透 | 延续 V0.2 的社区回复策略，保持每周 3-5 次有质量的社区互动 |
| 模型参数库共建 | 在 README 中明确标注"社区共建"，引导用户提 PR 贡献新模型参数。贡献者名字出现在 README 中 |
| Agent 框架集成准备 | 向 Hermes 提 PR，将秋毫mem 作为 MCP Tool 推荐配置写入文档 |
| V1.0 预告 | 发一篇技术博客《秋毫mem 的下一步：让 Agent 越住越了解自己》，预告动态校准功能 |

**市场判断节点**（V0.3/0.4 结束时）：

| 评估项 | 条件 | 决策 |
|--------|------|------|
| 用户反馈验证 | V0.2 反馈 Issue 收到 20+ 条有效反馈，且 80% 正面 | 继续 V1.0 |
| 竞品动态 | 是否有竞品推出类似功能？ | 若有，V1.0 开发周期压缩 30%，砍掉 Web 仪表盘 |
| 技术稳定性 | 守护进程 + 插件体系是否稳定？ | 若不稳定，V1.0 延后 2 周 |

### 四、风险预案

| 风险 | 触发条件 | 应对 |
|------|----------|------|
| V0.2 被竞品逆向 | Ollama/llama.cpp 推出类似功能 | 不恋战，缩短 V0.3/0.4 周期 30%，加速抵达 V1.0 |
| V1.0 引爆效果不达预期 | D+7 后 Stars 增长 < 300 | 启动付费推广（技术周报广告），找 UP 主合作 |
| 社区出现竞品对比帖 | 有人问"秋毫mem 和 XX 有什么区别" | 只讲自己的品类定位，不点评竞品，不参与对比 |
| 种子用户预演反馈不佳 | 5 个种子用户中 3 个觉得"不值得用" | 调整 V1.0 叙事，重新做预演，延迟发布 |
