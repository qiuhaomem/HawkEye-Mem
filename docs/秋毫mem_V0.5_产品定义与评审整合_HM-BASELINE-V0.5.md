## 一、版本概述

### 1.1 版本代号：「钓鱼行动」

V0.5 是秋毫mem从"独立工具"到"生态入口"的转折点。核心逻辑：

- **🪤 鱼饵**：极致缓存策略 Hermes Skill，一个高价值低门槛的引流入口
- **🎣 渔具**：秋毫mem `cache_strategy` 字段、MCP `get_cache_strategy`/`report_cache_hit` 工具
- **📣 自传播**：五个钓鱼暴露点 + 成本报告水印，用户截图自带秋毫mem名字

### 1.2 版本定位

| 维度 | V0.4 | V0.5 |
|------|------|------|
| 缓存策略 | 无（秋毫mem不参与） | 动态缓存策略，与内存压力联动 |
| 用户获取 | 靠社区自然搜索 | 靠缓存Skill主动引流 |
| 自传播 | 依赖用户自发 | 成本报告截图自带秋毫mem水印 |
| 与Agent框架集成 | Hermes MCP基础集成 | Hermes Skill深度集成，一键安装 |
| 缓存命中率 | 60-70%（静态） | 95-99.7%（动态调整） |

### 1.3 核心鸿沟

秋毫mem知道"当前内存压力是 low/medium/high/critical"，但不知道"这个压力下缓存策略该怎么调"。V0.5 填补这个鸿沟。

### 1.4 用户画像

- **主要用户**：每天用Hermes跑大量任务的开发者，API费用是主要成本
- **次要用户**：刚入门本地Agent的新手，被OOM困扰，听说有工具能省钱+防崩
- **传播节点**：在社区晒"我这个月省了多少API费用"截图的用户

## 二、五级需求分解表

### 应用端 1：极致缓存策略 Skill（鱼饵 🪤）

| 一级 | 二级 | 三级 | 四级 | 五级 |
|------|------|------|------|------|
| **缓存Skill** | **安装与引导** | **零配置启动** | 下载到 `~/.hermes/skills/hermes-cache-strategy/`，Hermes自动加载 | 无需编辑YAML，无需重启；安装后首次运行自动检测秋毫mem |
| | | **秋毫mem检测** | Skill启动时调用 `which hawk-eye-mem` | 未安装时输出引导文案；已安装时自动连接MCP |
| | | **引导文案** | 未检测到秋毫mem：提示命中率约60%，建议安装秋毫mem可达99%+ | 文案固定为三行：现状→目标→安装命令 |
| | | **自助安装模式（REQ-013-SUP-001）** | Skill首次运行检测到秋毫mem未安装时，引导用户确认一次，由Agent自动完成后续所有安装步骤 | 见附录A：自助安装模式 |
| | **缓存模式决策** | **动态模式切换** | 每次任务开始前，通过MCP调用秋毫mem获取 `cache_strategy` | 四种模式：aggressive/balanced/conservative/emergency |
| | | **降级运行** | 秋毫mem不可用时，回退到静态铁律（当前行为） | 命中率降至约60%，但不中断任务 |
| | | **模式触发条件** | Skill只读取秋毫mem的 `cache_strategy`，不自行判断 | 解耦设计 |
| | **成本报告输出** | **每次任务结束生成报告** | 任务完成后输出成本节省摘要 | 包含：总调用数、命中次数、命中率、节省金额、本月累计 |
| | | **报告底部水印** | 固定格式：动态缓存策略由秋毫mem提供 + 当前压力 + 缓存模式 + 安装命令 | 不可关闭，每份报告都有 |
| | | **报告格式** | 终端彩色框线输出，同时支持 `--json` 输出机器可读版本 | JSON格式供Agent消费 |
| | **安全与回退** | **紧急清空缓存** | 秋毫mem报告 `pressure: critical` 时，Skill立即清空L2/L3缓存 | 不清空L1（Provider控制），保存任务状态后暂停新请求 |
| | | **恢复后自动重建** | 压力恢复到 `medium` 以下时，Skill自动重启缓存预热 | 无需人工干预 |
| | | **用户手动覆盖** | 提供 `--cache-mode aggressive/balanced/conservative/emergency` 参数 | 手动模式优先于秋毫mem建议；解除手动后恢复自动 |

### 应用端 2：秋毫mem缓存策略输出（渔具 🎣）

| 一级 | 二级 | 三级 | 四级 | 五级 |
|------|------|------|------|------|
| **秋毫mem缓存输出** | **`cache_strategy` 字段** | **JSON结构** | `agent_guidance` 同级新增 `cache_strategy` 字段 | 包含：mode/ttl_seconds/max_cache_mb/current_cache_mb/hit_rate_24h/prefetch_enabled/reason |
| | | **mode: aggressive** | 触发条件：`pressure=low` 且可用内存>总内存×30% | TTL=600s，prefetch=true，max_cache=可用内存×20% |
| | | **mode: balanced** | 触发条件：`pressure=medium` 或可用内存15-30% | TTL=300s，prefetch=true，max_cache=可用内存×10% |
| | | **mode: conservative** | 触发条件：`pressure=high` 或可用内存5-15% | TTL=60s，prefetch=false，max_cache=可用内存×5% |
| | | **mode: emergency** | 触发条件：`pressure=critical` 或可用内存<5% | TTL=0s，prefetch=false，max_cache=0，立即清空 |
| | | **hit_rate_24h 统计** | 从Skill汇报的缓存命中数据中计算 | Skill每次任务结束向秋毫mem汇报命中/未命中计数 |
| | | **reason 字段** | 人类可读的判断理由 | 例："内存充裕（可用4.2GB），启用激进缓存模式，预计命中率99%+" |
| | **MCP Server 增强** | **新增 `get_cache_strategy` 工具** | MCP Tool定义：输入可选 `model_name`，输出 `cache_strategy` JSON | 与现有 `check_memory` 工具共存 |
| | | **工具描述** | "获取当前系统资源状态下推荐的最佳缓存策略" | 供Agent和Skill调用 |
| | **数据埋点** | **Skill 汇报接口** | MCP 新增 `report_cache_hit` 方法 | Skill任务结束后汇报：`model_name/hit_count/miss_count/cost_saved` |
| | | **数据存储** | 追加到 `~/.config/hawk-eye-mem/cache_stats.jsonl` | 本地存储，不上传；保留30天 |
| | **与并发度联动** | **缓存策略影响并发度** | 当 `cache_strategy.mode=emergency` 时，`concurrency_guidance.max_safe_agents` 强制为0 | 缓存和并发共用底层内存池 |

## 三、多方评审会议纪要

**评审对象**：V0.5 产品调研报告 + 五级需求分解表
**会议主持**：高级产品经理
**参与方**：技术负责人、安全专家、市场运营、法务、PMO
**创始人列席**
**结论性质**：通过，含 8 条整改要求

### 👤 技术负责人

需求方向没问题，Skill与秋毫mem的交互逻辑清晰。但实现上有几个点必须现在敲定：

1. **Skill侧30秒缓存`cache_strategy`结果**，避免每次冷启动秋毫mem（~50ms），30秒内数据不会剧烈变化
2. **`report_cache_hit`设计为fire-and-forget**，失败不阻塞任务，不重试，仅记录debug日志
3. **`concurrency_guidance`和`cache_strategy`共享底层资源池**，避免各自算一遍（内部重构，~0.5天）
4. **确认Hermes运行时TTL修改可行性**，降级方案为Skill侧模拟TTL（标记时间戳，超时不用）

**结论**：通过。

### 👤 安全专家

1. **`mode=emergency`只暂停API调用，不影响其他工具调用**（文件读写、代码执行等），Skill只控制缓存行为
2. **缓存Stats模型名沿用V0.3的哈希脱敏策略**，防止统计文件泄露模型使用模式
3. **引导文案增加备用安装方式**（curl二进制下载），不依赖Homebrew

**结论**：通过。

### 👤 市场运营

1. **Skill文案支持中英文**，中文重写非直译，符合中文开发者语感
2. **成本报告水印上方加一行"📊 此报告由秋毫mem提供技术支持"**，用emoji吸引视线，截图不易被裁掉
3. **推广素材统一写"24小时缓存命中率99.7%"**——多三个字，可信度翻倍

**结论**：通过。

### 👤 法务

1. **DISCLAIMER增加缓存自动调整风险条款**：用户启用Skill自动模式即表示接受缓存自动调整可能带来的上下文丢失风险
2. **成本报告增加"节省金额为估算值，实际费用以API提供商的账单为准"声明**

**结论**：通过。

### 👤 PMO

1. **V0.4发布后2周内启动V0.5**，P0模块优先
2. **W1冻结MCP接口定义**（`get_cache_strategy`和`report_cache_hit`的输入输出格式），Skill和秋毫mem端并行开发
3. **编写自动化循环测试脚本**：100个模拟任务，验证缓存命中率统计准确、暴露点文案触发正确

**结论**：通过。

## 附录A：Agent自助安装模式（REQ-013-SUP-001）

**增补编号**：REQ-013-SUP-001
**增补事由**：创始人要求引导用户确认让Agent自己安装curl、Skill、MCP等所需组件
**增补位置**：应用端1（缓存Skill）→ 安装与引导 → 新增"自助安装模式"

### A.1 自助安装流程

```
Skill启动 → 检测秋毫mem → 未安装
  → 输出引导："秋毫mem能帮你省90%API费用，装吗？[Y/n]"
  → 用户输入Y
  → 检测系统环境（brew可用？→ 用brew；不可用 → 用curl二进制）
  → 执行安装 → 验证 → 配置MCP → 验证MCP连接
  → 输出："装好了，以后每次任务我都用它帮你省钱。"
  → Skill以完整模式运行
```

### A.2 交互式引导文案

```
[hermes-cache-strategy] 秋毫mem未安装，缓存命中率约60%
[hermes-cache-strategy] 安装秋毫mem后可达99%+，帮你省API费用
[hermes-cache-strategy] 是否允许我帮你自动安装？[Y/n]
→ 用户输入Y
[hermes-cache-strategy] 正在安装秋毫mem...
[hermes-cache-strategy] brew install hawk-eye-mem... ✅ 完成
[hermes-cache-strategy] 验证安装... ✅ hawk-eye-mem v0.4.0
[hermes-cache-strategy] 配置MCP Server... ✅
[hermes-cache-strategy] 验证MCP连接... ✅
[hermes-cache-strategy] 秋毫mem已就绪，缓存命中率目标99%+
[hermes-cache-strategy] 以后每次任务我都会用它帮你省钱。
```

### A.3 安装失败处理

| 步骤 | 失败时输出 |
|------|-----------|
| brew install | "brew安装失败，请手动执行: brew install hawk-eye-mem，或访问 https://github.com/qiuhaomem/hawk-eye-mem 下载二进制。" |
| curl下载 | "curl下载失败，请手动访问 https://github.com/qiuhaomem/hawk-eye-mem/releases 下载最新二进制。" |
| MCP配置 | "MCP配置失败，请手动编辑Hermes配置添加秋毫mem MCP Server，详见: https://github.com/qiuhaomem/hawk-eye-mem" |

### A.4 安全约束

- Agent仅执行安装命令，不执行任何其他shell命令
- 安装命令限定白名单：`brew install`、`curl`下载到指定路径、`chmod`、`mv`到`/usr/local/bin`
- 安装脚本不包含`sudo`，若需要`sudo`（如写入`/usr/local/bin`），Agent提示用户手动执行最后一步
- 所有安装命令执行前打印到终端，用户可审查
- 安装全程日志可见，用户可随时Ctrl+C中断
- 中断后Skill降级为静态铁律运行

### A.5 对现有需求的影响

| 受影响项 | 变更前 | 变更后 |
|----------|--------|--------|
| 首次运行暴露点文案 | 输出三行静态文案让用户手动安装 | 交互式引导，用户确认后Agent自动安装 |
| 安装门槛 | 用户需自己敲`brew install` | 只需输入Y，Agent代劳 |
| 用户记忆点 | "这工具让我装个东西" | "这Agent自己把环境装好了" |
| 备用安装方式（CR-07） | 文案中提示curl备用 | Agent自动判断brew不可用时走curl路径 |
