## 二、技术可行性评估

### 2.1 数据源解析

| 数据源 | 格式 | 解析方式 | 难度 | 风险 |
|--------|------|---------|------|------|
| state.db | SQLite | `rusqlite` 或 Python `sqlite3` | ⭐⭐ | Schema 随 Hermes 版本变化，需做版本兼容 |
| agent.log | 文本 | 正则匹配 | ⭐ | 日志格式可能随版本调整，正则需宽松匹配 |
| crontab | 文本 | `crontab -l` 解析 | ⭐ | 用户可能无 crontab 权限 |
| Provider 定价 | 硬编码 + 配置文件 | 内置定价表 + 用户自定义 | ⭐ | 定价需定期更新 |

**关键决策**：Token 审计作为 Skill 独立运行，用 Python 实现。数据解析本地完成，不上传任何数据。

### 2.2 技术实现路径

```
Token审计Skill 启动
  ├─ 检测数据源可用性
  │   ├─ state.db → ✅ 可用 / ❌ 不可用（提示用户）
  │   ├─ agent.log → ✅ 可用 / ❌ 不可用
  │   └─ crontab → ✅ 可用 / ❌ 无权限
  ├─ 数据采集
  │   ├─ 解析 state.db（SQL查询）
  │   ├─ 解析 agent.log（正则匹配）
  │   └─ 解析 crontab（文本解析）
  ├─ 审计分析
  │   ├─ 总账计算
  │   ├─ 来源分布
  │   ├─ 浪费检测
  │   ├─ cron审计
  │   └─ 费用对比
  └─ 报告生成
      ├─ 一句话总结
      ├─ 结构化报告（终端彩色输出）
      └─ JSON输出（可选）
```

### 2.3 关键代码模块

| 模块 | 语言 | 预估代码量 | 依赖 |
|------|------|-----------|------|
| state.db 解析器 | Python | ~150 行 | `sqlite3`（内置） |
| agent.log 解析器 | Python | ~100 行 | 无 |
| crontab 解析器 | Python | ~50 行 | 无 |
| 审计分析引擎 | Python | ~200 行 | 无 |
| 报告生成器 | Python | ~150 行 | 无 |
| CLI 入口 | Python | ~80 行 | `argparse`（内置） |
| **总计** | | **~730 行** | 零外部依赖 |


## 三、测试策略

### 3.1 用例分布

| 测试类型 | 用例数 | 覆盖内容 |
|----------|--------|----------|
| 单元测试 | 25 | state.db 解析、agent.log 解析、crontab 解析、审计计算、报告生成 |
| 集成测试 | 10 | CLI 参数组合、JSON 输出格式、与缓存策略联动 |
| 用户体验测试 | 8 | 一句话总结可读性、报告美观度、双饵互引链接 |
| 边界/异常测试 | 8 | state.db 不存在/损坏、agent.log 超大、无 crontab 权限 |
| **总计** | **51** | |

### 3.2 关键验收场景

| 场景 | 验收标准 |
|------|----------|
| 正常审计 | 输出一句话总结 + 结构化报告，数据与 state.db 一致 |
| 无 state.db | 提示"未找到 Hermes 数据库"，不崩溃 |
| agent.log 超大（>10MB） | 只读末尾 10MB，不卡死 |
| 无 crontab 权限 | 跳过 cron 审计，报告中标注 |
| JSON 模式 | `--json` 输出合法 JSON，可被 `jq` 解析 |
| 与缓存策略联动 | 审计报告中出现"缓存策略帮你节省了 $X" |

## 五、风险评估与应对

| 风险 | 等级 | 应对 |
|------|------|------|
| Hermes state.db Schema 变化 | 中 | 查询前检测 Schema 版本，不匹配时提示用户升级 |
| agent.log 格式不稳定 | 低 | 正则匹配宽松，匹配不到时标注"无法识别"而非报错 |
| 用户无 crontab 权限 | 低 | 静默跳过，报告中标注"无权限" |
| 用户不清楚"Token审计是什么" | 中 | 一句话总结降低认知门槛；缓存报告底部引导 |
| 两个 Skill 并行开发资源不足 | 中 | Token 审计可降级为缓存策略的内置报告模块，而非独立 Skill |
