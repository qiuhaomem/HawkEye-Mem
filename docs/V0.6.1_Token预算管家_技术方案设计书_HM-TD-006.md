# 秋毫mem V0.6.1 · Token预算管家 技术方案设计书

## 一、概述

### 1.1 背景

Hermes Agent 运行过程中产生大量 Token 消耗，用户缺乏可视化手段了解自己的 API 费用到底花在哪里、浪费了多少。V0.5 Token 审计解决了"查账"问题，V0.6.1 的 Token 预算管家更进一步：**持续监控 → 识别浪费 → 给出建议 → 一键执行**。

### 1.2 目标

- 用户能通过 `hawk-eye-mem --token-budget` 一眼看清自己的 Token 消耗和费用
- 自动识别 6 种典型浪费场景，给出预估可节省费用
- 优化建议支持 `--dry-run` 预览 + 一键执行 + 一键回滚

### 1.3 非目标

- 不做实时限流/限速（Agent 框架自己的事）
- 不做 Provider 账单对账（边界外）
- 不做跨用户 Token 分摊（V1.0 后再考虑）

## 二、架构设计

### 2.1 模块结构

```
src/budget/
├── mod.rs              # 模块导出
├── collector.rs        # Token 数据采集（state.db + agent.log）
├── analyzer.rs         # 浪费分析和优化建议
├── cost.rs             # 费用换算（定价表）
├── executor.rs         # 优化执行器（配置修改/备份/回滚）
├── report.rs           # 报告生成
```

### 2.2 CLI 接口

采用 `--token-budget` + 子动作模式（复用 `clap` 已有架构）：

```
hawk-eye-mem --token-budget status          # Token 消耗总览
hawk-eye-mem --token-budget waste           # 浪费分析
hawk-eye-mem --token-budget suggest         # 优化建议
hawk-eye-mem --token-budget apply <ID>      # 执行优化（配合 --dry-run）
```

### 2.3 MCP 工具

| 工具名 | 功能 | 输出 |
|--------|------|------|
| `get_token_budget_status` | 当前 Token 消耗状态 | JSON（摘要 + 费用估算） |
| `get_token_budget_suggestions` | 优化建议列表 | JSON（建议列表 + 预期节省） |
| `run_token_budget_optimization` | 执行优化 | JSON（执行结果 + 备份路径） |

## 三、详细设计

### 3.1 Token 数据采集器 (`collector.rs`)

**数据源优先级**：

```
1. Hermes state.db          # 最准确：含实际 tokens / cache / cost
2. Hermes agent.log         # 补充：API call 级别的日志
3. 降级：数据源不可用时输出提示
```

**state.db 解析**：
- 路径：`~/.hermes/state.db`
- 表：`sessions`（字段：`input_tokens`, `output_tokens`, `cache_read_tokens`, `cache_write_tokens`, `estimated_cost_usd`, `provider`, `model`, `created_at`）
- 查询：`SELECT * FROM sessions WHERE created_at >= datetime('now', '-24 hours')`

**agent.log 解析**：
- 路径：`~/.hermes/logs/agent.log` + `agent.log.1`
- 格式：JSON Lines
- 关键字段：`in`, `out`, `cache`, `latency`, `model`, `provider`
- 提取：正则匹配 `API call #N: model=... in=... out=... cache=... latency=...`

**数据聚合**：
- 按模型聚合
- 按时段聚合（最近 1h / 24h / 7d）
- 按会话聚合（首轮 vs 后续）

### 3.2 浪费分析引擎 (`analyzer.rs`)

**6 种浪费场景检测**：

| 场景 | 检测条件 | 严重度 | 预估节省 |
|------|---------|:------:|:--------:|
| 冷启动过大 | 首轮 input > 30K tokens | 🔴 高 | 30-50K tokens/次 |
| 缓存命中率低 | cache_hit_rate < 50% | 🔴 高 | 40-60% tokens |
| 冗余技能注入 | 技能列表但从未被引用 | 🟡 中 | 5-10K tokens/轮 |
| Memory 臃肿 | memory 注入 > 5K tokens | 🟡 中 | 3-8K tokens/轮 |
| 冗余 MCP Server | 有 Server 但未被调用 | 🟢 低 | 5-15K tokens/启动 |
| 上下文窗口浪费 | 大上下文但只用了小部分 | 🟡 中 | 10-30K tokens/轮 |

**评分算法**：
```
waste_score = (estimated_waste_tokens / total_tokens) * 100
savings = estimated_waste_tokens * cost_per_token
priority = based on severity + savings magnitude
```

### 3.3 费用换算器 (`cost.rs`)

**内置定价表**（编译期嵌入 JSON）：

```json
{
  "deepseek/deepseek-v4-flash": {
    "input_per_1M": 0.07,
    "output_per_1M": 0.28,
    "cache_hit_per_1M": 0.014
  },
  "anthropic/claude-sonnet-4": {
    "input_per_1M": 3.00,
    "output_per_1M": 15.00,
    "cache_hit_per_1M": 0.30
  }
}
```

**用户自定义定价**：`~/.config/hawk-eye-mem/custom_prices.json`，优先级高于内置。

### 3.4 优化执行器 (`executor.rs`)

**安全机制**：

```
apply <ID>
  ├─ 1. 读取建议ID对应的优化动作
  ├─ 2. 备份当前配置 → ~/.config/hawk-eye-mem/backups/<timestamp>/
  ├─ 3. --dry-run: 显示 diff，不实际修改
  ├─ 4. 实际执行：修改 Hermes config.yaml / 技能禁用列表 / 缓存配置
  ├─ 5. 验证：修改后是否生效
  └─ 6. 支持 --rollback: 恢复到最近的备份
```

**可执行的优化类型**：

| 类型 | 操作 | 目标文件 | 回滚方式 |
|------|------|---------|---------|
| 禁用技能 | 添加 skills.disabled 条目 | `~/.hermes/config.yaml` | 删除条目 |
| 调整缓存 | 修改缓存 TTL / 模式 | `~/.config/hawk-eye-mem/config.toml` | 恢复备份 |
| 压缩 memory | 截断或归档 memory 文件 | `~/.hermes/memories/MEMORY.md` | 恢复备份 |
| 移除 MCP | 删除未使用的 MCP Server | `~/.hermes/config.yaml` | 重新添加 |

## 四、数据流

```
agent.log  →  [LogParser]  →  TokenRecords
                                    ↓
state.db   →  [DBReader]   →  TokenRecords  →  [Aggregator]  →  TokenSummary
                                                                    ↓
                                                           [WasteAnalyzer]
                                                                    ↓
                                                           WasteReport[]  →  CLI 输出
                                                                    ↓
                                                           [CostCalculator]
                                                                    ↓
                                                           SavingsEstimate
                                                                    ↓
                                                           [SuggestionGenerator]
                                                                    ↓
                                                           Suggestion[]  →  apply <ID>
                                                                    ↓
                                                           [Executor]
                                                           ├─ backup
                                                           ├─ modify
                                                           └─ verify
```
