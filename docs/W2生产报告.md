## 三、新增/变更模块

### 3.1 新增：GuidanceGenerator（`src/engine/guidance.rs`）

| 元素 | 说明 |
|:----|:-----|
| `Action` 枚举 | `Ok` / `Monitor` / `ReduceContext` / `AbortSafely` |
| `Guidance` 结构体 | `pressure` + `action` + `reason` + `suggestion` + `estimated_safe_context_window` + `confidence` |
| `GuidanceGenerator::generate()` | 从内存指标生成完整 Agent 建议 |
| `GuidanceGenerator::classify()` | 纯函数，4级压力分类（low/medium/high/critical）|

**JSON输出示例**：
```json
{
  "agent_guidance": {
    "action": "monitor",
    "confidence": "conservative",
    "estimated_safe_context_window": 1710284,
    "pressure": "medium",
    "reason": "Moderate: 4772MB available, 39.3% used. Continue monitoring.",
    "suggestion": "Moderate memory pressure. Consider configuring model parameters for better accuracy."
  }
}
```

### 3.2 移除：`calc_pressure` 函数

原 `main.rs` 中的内联压力计算已由 `GuidanceGenerator::classify()` 替代，消除代码重复。

### 3.3 跨平台兼容性修复

| 问题 | 修复方式 |
|:----|:---------|
| Linux collector 在 macOS 上死代码警告 | 条件编译 + `#[allow(dead_code)]` |
| macOS collector 在 Linux 上死代码警告 | 同上 |
| `CollectError` 变体平台专属 | 逐字段 `#[allow(dead_code)]` |
| 集成测试 unused import | 清理 `std::thread` 和废弃的 `onboarded_path()` |

## 六、遇到的问题与解决方案

| 问题 | 影响 | 解决方案 |
|:----|:-----|:---------|
| GitHub Actions `-D warnings` 将跨平台死代码告警转错误 | macOS/Linux CI 均失败 | 添加 `#[cfg_attr(not(...), allow(dead_code))]` 条件编译 |
| 本地 `.cargo/config.toml` 自定义链接器在CI上不存在 | Linux CI 链接失败 | 移除仓库中的 `.cargo/` 目录，CI使用默认链接器 |
| YAML内嵌Python脚本转义问题 | Workflow语法错误 | 改用 `jq` 处理JSON，纯shell验证 |
| 首次运行测试并行冲突 | `.onboarded` 文件状态竞争 | 使用独立临时HOME目录，合并为单测试函数 |
| DeepSeek-TUI API key 与 Hermes 不一致 | TUI认证失败 | 待统一API凭据配置（今晚处理） |
| Reasonix 文件写入未落盘 | Reasonix 模拟写入不生效 | 待确认正确调用方式 |
