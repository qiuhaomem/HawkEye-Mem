## 四、技术架构基准

### 4.1 模块结构（V0.6 更新）

```
src/
├── main.rs              # CLI 入口 + 参数解析
├── config.rs            # 配置文件读取
├── models.rs            # 数据模型
├── helpers.rs           # 工具函数
├── suggest.rs           # 并发度建议
├── thermal.rs           # 温度监控
├── multi_agent.rs       # 多Agent检测
├── cache/               # 缓存策略模块
│   ├── mod.rs
│   ├── advisor.rs       # 缓存策略建议器
│   └── stats.rs         # 缓存统计（V0.6增强：差距分析）
├── calibration/         # 校准引擎
├── collector/           # 数据采集器
├── commands/            # CLI 命令处理
├── container/           # 容器适配
├── engine/              # 评估引擎
├── environment/         # 环境指纹
├── gpu/                 # GPU 监控
├── remote/              # 远程采集
├── state_machine/       # 状态机
└── trends/              # 趋势分析
```

### 4.2 测试策略

- 单元测试：每个模块内 `#[cfg(test)]`
- 集成测试：`tests/` 目录，模拟不同内存压力场景
- 目标：V0.6 ≥ 35 tests（当前 28）
